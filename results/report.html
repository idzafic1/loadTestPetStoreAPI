Understanding Load Testing

Response Time (Latency)

Time between sending a request and receiving a response. Lower is better. Measured in milliseconds (ms).

P95 / P90 (Percentiles)

P95 means 95% of requests were faster than this value.Shows worst-case performance excluding outliers.

Error Rate

Percentage of requests that failed (non-2xx responses).Should be as close to 0% as possible.

RPS (Requests Per Second)

Throughput - how many requests the API can handle per second.Higher is better.

Virtual Users (VUs)

Simulated concurrent users making requests.More VUs = higher load on the system.

Iterations

Complete test script executions.Each iteration runs through all defined requests.

Test Types Explained

Smoke Test

Minimal load test to verify basic functionality.

  • 1 virtual user
  • Duration: 1 minute
  • Purpose: Sanity check

Stress Test

Gradually increases load to find breaking point.

  • Ramps to 100 users
  • Duration: 5.5 minutes
  • Purpose: Find limits

Spike Test

Sudden traffic surge to test recovery.

  • 5 to 100 users suddenly
  • Duration: 2.5 minutes
  • Purpose: Test resilience

Full Load Test

Comprehensive test with all scenarios.

  • All 4 scenarios
  • Duration: 8 minutes
  • Purpose: Complete analysis
